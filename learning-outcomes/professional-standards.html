Blik Theatre Stylescape

The Blik Theatre's stylescape attempted to portray the essence of the theater by combining professionalism, emotion, and playfulness into a cohesive brand identity.

- Situation: Blik Theatre required an original rebranding, or maybe just some branding in general, that mirrored its function as a playback theatre for public performances and team-building exercises.

- Task: I had to design a stylescape that reflected the theater's basic principles of inclusivity, creativity, and connection.

- Action: I looked at color schemes, typography, and imagery through my research. I incorporated human-focused graphics to emphasize narrative and teamwork, and I used strong typefaces and warm, brilliant colors to communicate liveliness and connection.

- Result: My finished stylescape was well received within the team, but we ended up choosing someone else's stylescape by vote.

- Reflection: This assignment strengthened my ability to communicate emotions visually and the ability to match a brand's emotional core with its design.

Github Version Control

In my opinion using Github for version control, especially in group project, is essential to properly keep track of your progress. That's why I used GitHub for version control during the semester, which allowed for efficient teamwork, progress monitoring, and workflow organization. This experience has a clear connection to the learning outcome: Development, but also with Professional Standards as it demonstrates my proficiency with industry-standard tools for well-organized and productive coding procedures.

- Situation: We needed a solution to easily track changes, manage our code collectively, and make sure that our work remained error-free and well-organized for one of our group projects. To meet these requirements, I suggested using version control with GitHub.

- Task: It was my responsibility to set up GitHub as our project's version control system, working with other team members to ensure appropriate branching, committing, and merging procedures.

- Action:

  • I made the project's structure, including images, code, and documentation folders, and created a GitHub repository.
  • To prevent conflicts in the main branch, I made distinct branches for each team member to work on specific elements.
  • To track progress, frequent commits with accurate commit messages were recommended.
  • I made made sure all code passed testing before merging into the main branch, handled merge conflicts when they occurred, and reviewed changes using pull requests during integrating.
  • I also gave my teammates a quick lesson on how to use GitHub efficiently, going over the fundamentals like cloning, pushing, and pull requests.

- Result: Our project was made easier by using GitHub. It made sure that everyone had access to the most recent version of the project and made it possible for us to swiftly find and fix any problems. My team gave me good feedback because Github made it simpler to monitor progress and record changes.

- Reflection: My knowledge of version control and sharing code improved as a result of using GitHub. It confirmed the significance of upholding proper steps and good us of comments within a group. It shows my ability to use industry standard tools that enable effective, professional-level coding, which is in line with the Development learning outcome.

Research UX Websites

I researched the usability and user experience of Zekur's current website while working on the project. This required determining the problems that prevented users from effectively understanding and finding the insurance options. In line with my learning outcome of professional standards, the result served as the basis for developing a more efficient and user-friendly website.

- Situation: Analyzing and enhancing Zekur's insurance website's user experience was part of the project. My objective was to improve the platform's usability and accessibility for as many users as possible.

- Task: It was my responsibility to find usability problems and suggest fixes. To improve the user experience overall, this involved looking at the site's layout, navigation, and content clarity.

- Action: To do this, I thoroughly researched and assessed the website to identify any problems. The primary issues were unclear buttons, a home screen reserved for freelancers, and unnecessary scrolling due to large letters. I also evaluated how hard it was to access important features like "Calculate Premium" and how few options there were to customize for extra insurance. I brainstormed several solutions, such as revamping the navigation, increasing the visibility of buttons, and adding features like summaries of combined insurance benefits.

- Result: I was able to redesign the website with an emphasis on better user experience and navigation thanks to the research findings. A unmissable "Continue" button, a smaller font size for simpler browsing, and simplified navigation are some of the modifications I made. U can see more changes on the actual wireframes themselves. The website's usability and clarity were greatly improved by these changes.

- Reflection: This task shows how important testing happy flows and possible errors users might run in to are to producing successful designs. Delivering a polished and user-friendly product required recognizing and resolving pain areas.

Group Contract

Setting up clear agreements within a team to guarantee productive cooperation is another aspect of upholding professional standards. Our team created a comprehensive studio contract that included roles, communication guidelines, and standards for accountability, attendance, and feedback.

- Situation: We required a clear contract for our group project in Semester 3 in order to organize our team and guarantee that each member made a meaningful contribution to the project's objective. This was especially important because the project needed weeks of working together.

- Task: It was our responsibility to draft a studio contract that would guarantee accountability and uphold professional standards throughout the project by defining roles, communication channels, and responsibilities.

- Action: Together, we created a studio contract that included the following terms for our studio, GraphicAlley(later renamed to Patched):

  • Platforms for communication: Trello for scheduling and planning, Teams channel for sharing documents, and Teams chat for rapid conversation.
  • Roles: To define duties, each member was given a role, such as developer, strategist, or designer.
  • Strikes and Attendance: We implemented a strike mechanism for accountability along with guidelines for managing absences and attendance.
  • Planning and Responsibilities: Group feedback was routinely recorded in Feedpulse, and weekly planning meetings were scheduled for Mondays at 10:00.
  • Roles: To define duties, each member was given a role, such as developer, strategist, or designer.
    • Additionally, we stressed the importance of open communication, offering constructive criticism, and establishing a safe atmosphere for learning. To officially declare our agreement, the contract was signed by each member after it was completed.

- Result: Our team has a strong basis thanks to the studio contract. We were able to concentrate on project requirements, guarantee accountability, and minimize misunderstandings.

- Reflection: It made it clear how important precise agreements and organized teamwork are to upholding professional standards. We increased our productivity and made sure that each member made a significant contribution to the project by creating and following the studio contract.

Research AI Fairytales

It takes good research into a number of topics, including reading levels, attention spans, and creative storytelling techniques, to determine how to make fairytales interesting, age-appropriate, and digestible for kids. Along with exploring the effects of blue light on kids, this research offered creative substitutes for screen-based storytelling. I learned a lot about how to match methods of storytelling to technology through this approach, which directly relates to the Learning Outcome for Professional Standard.

- Situation: In order to investigate different approaches of presenting fairytales to children, I had to do some research. Finding creative, age-appropriate solutions that didn't use to much blue light since kids tend to listen to fairytales before sleeping and that might reduce their quality of sleep was the aim. But I also have to make sure the information is interesting and usable by kids at all developmental stages.

- Task: My task was to gather information on:

  1. Alternatives to conventional screen-based narrative that emphasize interactive and immersive techniques.
  2. The consequences of blue light exposure for kids, including how it affects their development and health.
  3. The attention span of kids in different age groups to make sure the suggested solutions are fit for their mental capacities.
  4. Reading levels using the AVI system, to tailor content for different age ranges.

- Action:

  1. Exploring Alternatives:
    • I came up with ideas for imaginative methods to narrate fairy tales, which led to AR glasses, interactive pillows, nightlight storytellers, and plush toys that tell stories.
    • I evaluated how each idea would improve participation while resolving issues with screen addiction and blue light.
  2. Researching Blue Light Effects:
    • I investigated how blue light affects children's sleep patterns, eye strain, and cognitive development.
    • I thought of some practical solutions, such as blue light filters and reducing screen time before bed.
  3. Studying Attention Spans:
    • I researched some studies on kids' attention spans at various ages, looking at how degrees of involvement varied and how interaction can help kids stay focused.
    • This influenced suggestions for age-based storytelling forms and lengths.
  4. Understanding AVI Reading Levels:
    • In order to match fairytale content with the reading levels of various age groups, I looked into the AVI reading system.
    • I suggested the storytelling techniques should enhance young readers' reading progress without being overbearing.

- Result: This research offered a broad viewpoint on how to effectively change fairytales for todays children. The suggested solutions address health issues like blue light exposure while integrating immersive technology with developmental factors like reading levels and attention spans. This guarantees a well-rounded strategy that is both creative and kid-friendly.

- Reflection: I learned from this research just how important it is to incorporate various points of view when creating solutions for kids, including those connected to technology, development, and health. This task also emphasized how important it is to base your concept ideas on properly researched data in order to guarantee their practicality and impact.

Learning Next.js, Typescript and Tailwind with Cursor AI

The creation of the "Skillz" student portfolio system for sprint X was my last project. I used modern web development tools like Next.js, TypeScript, and Tailwind CSS in this project. AI technologies like Claude and Cursor, which offered help and explanations while I developed the program, improved this experience. I discovered how these technologies cooperate to produce reliably functioning and type-safe web apps by developing this application.

- Situation: In order to track student projects across multiple subjects (Metaal, Hout, Bouw, and AT), I had to create a web application. Although I knew the fundamentals of programming, I was unfamiliar with Next.js, TypeScript, and Tailwind CSS.

- Task: My goals were to:

  • Create a modern online application by learning and utilizing Next.js.
  • To provide type safety to JavaScript, learn TypeScript.
  • Use Tailwind CSS for efficient styling.

- Action:

  1. Using Cursor AI:
    • Learned the correct TypeScript syntax by using Cursor's code completion.
    • Real-time recommendations for Next.js component structures were received.
    • Received instant feedback on recommended practices and possible mistakes.
  2. Using Claude:
    • Requested clarifications on things that I didn't know.
    • Obtained thorough code structure breakdowns
    • Received assistance with error resolution and debugging.
  3. Practical Implementation:
    • Next.js was used to create a file-based routing system.
    • Using Typescript to make myself follow proper coding practices.
    • Tailwind CSS was used to create responsive layouts.
    • Created reusable components such as StudentCard and ProjectCard.

- Result: I created a functional student portfolio application with success and acquired hands-on experience in:

  • Next.js file-based routing and component structure.
  • How to use TypeScript.
  • Tailwind CSS utility classes and responsive design.
  • Modern web development best practices.

- Reflection: The combination of AI-tools proved helpful for learning:

  • Cursor AI provided immediate coding assistance and error prevention.
  • Claude offered detailed explanations and helped understand the "why" and “how” behind the code.
  • The hands-on project approach helped understanding and remembering the knowledge instead of watching hour long tutorials on youtube.

The most significant learning moments came when:

  1. Understanding the difference between Next.js server and client components.
  2. Using TypeScript's type system through practical examples.
  3. Learning Tailwind's utility-first approach to CSS, though this was easier than expected since most of it is quite recognizable from standard CSS.

This project showed how AI-tools can accelerate learning while building real-world applications. The project evolved from basic concepts to implementing more complex features like subject filtering and year-based project organization, all while maintaining type safety and clean code practices and good habits.
