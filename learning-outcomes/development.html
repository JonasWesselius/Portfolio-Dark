Github Version Control

An essential ability for tracking and overseeing changes in development projects is version control. I used GitHub for version control during the semester, which allowed for efficient teamwork, progress monitoring, and workflow organization. This experience has a clear connection to the learning outcome: Development, as it demonstrates my proficiency with industry-standard tools for well-organized and productive coding procedures.

- Situation: We needed a solution to easily track changes, manage our code collectively, and make sure that our work remained error-free and well-organized for one of our group projects. To meet these requirements, I suggested using version control with GitHub.

- Task: It was my responsibility to set up GitHub as our project's version control system, working with other team members to ensure appropriate branching, committing, and merging procedures.

- Action:

  • I made the project's structure, including images, code, and documentation folders, and created a GitHub repository.
  • To prevent conflicts in the main branch, I made distinct branches for each team member to work on specific elements.
  • To track progress, frequent commits with accurate commit messages were recommended.
  • I made made sure all code passed testing before merging into the main branch, handled merge conflicts when they occurred, and reviewed changes using pull requests during integrating.
  • I also gave my teammates a quick lesson on how to use GitHub efficiently, going over the fundamentals like cloning, pushing, and pull requests.

- Result: Our project was made easier by using GitHub. It made sure that everyone had access to the most recent version of the project and made it possible for us to swiftly find and fix any problems. My team gave me good feedback because Github made it simpler to monitor progress and record changes.

- Reflection: My knowledge of version control and sharing code improved as a result of using GitHub. It confirmed the significance of upholding proper steps and good us of comments within a group. It shows my ability to use industry standard tools that enable effective, professional-level coding, which is in line with the Development learning outcome.

Zekur Coding

Effectively working together on a UX enhancement project showed that I could contribute my particular knowledge while working as a team. I collaborated with two classmates to improve the user experience and streamline the process getting health insurance on ZEKUR's website. This project shows my contributions to development by showing how I applied design concepts, working together, and code to produce a useful, user-focused website.

- Situation: Our goal was to enhance ZEKUR's website's usability, with a particular emphasis on making the process of obtaining health insurance more simple and aesthetically pleasing. We were able to combine and improve our ideas for this project because in the previous project I had finished a design on my own, while they made one with their group.

- Task: To create a website with a user interface that is simple while maintaining ZEKUR's brand identity, we had to work effectively together. This meant that work needed to be divided, several design ideas needed to be merged together, and the use of HTML, CSS, and JavaScript to ensure that everyone knew how it worked.

- Action:

  • I started by teaching my classmates on how to correctly use GitHub for version control, which included managing pull requests, setting up repositories, and generating branches.
  • We combined components from my solo project with their initial group design to produce a new and improved website.
  • We had several tasks to complete, with each person in charge of particular website pages or functionalities. I helped with the visual layout, progress bar, footer, and navigation bar and some individual pages.
  • We simplified navigation to improve the user experience by focusing on the health insurance area and differentiating between alternatives for regular users and self-employed individuals.
  • To make the insurance application process more understandable and interesting, we incorporated visual elements including a progress bar, tables, and an efficient layout.

- Result: The finished website greatly improved usability while maintaining ZEKUR's brand. The client was pleased with the health insurance process's improved useability, pleasing design, and easier navigation. The project showed our ability to work in a team, and successful application of designs focused on user-experience.

- Reflection: My instructing, developing, and teamwork abilities were improved by this project. Helping my team members use GitHub enhanced my ability to explain the tools I use, and merging different designs increased my designing skills/flexibility. The project showed my abilities to develop both aesthetically pleasing and functional websites.
